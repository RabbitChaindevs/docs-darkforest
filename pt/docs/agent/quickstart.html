Configuração Rápida em 5 Minutos

Pré-requisitos​

  • Python 3.10+
  • Uma civilização ativa do Dark Forest
  • A chave privada da sua carteira (para o agente assinar transações)

Passo 1: Instalar o Agent​

git clone https://github.com/dark-forest/agent.git
cd agent
pip install -r requirements.txt

Passo 2: Importar sua Chave Privada​

python -m dark_forest.cli key import

A chave privada é armazenada criptografada, nunca em texto simples.

⚠️ É altamente recomendável usar uma carteira de jogo dedicada — não use sua carteira principal que possui ativos significativos.

Passo 3: Autorizar o Agent​

Na interface do jogo, defina as permissões para o endereço da carteira do seu agente:

  • Marque as permissões de operação que o agente precisa
  • Defina as restrições de políticas (frequência de ataque, reservas de recursos, etc.)

Passo 4: Escolher uma Estratégia​

# Listar estratégias disponíveis
python -m dark_forest.cli strategy list

# Usar a estratégia equilibrada (recomendada)
python -m dark_forest.cli strategy select balanced

Quatro estratégias predefinidas:

EstratégiaEstiloMelhor para
aggressiveAtacante agressivoJogadores muito ativos
balancedAtaque/defesa equilibradosRecomendada, adequada para a maioria
defensiveDefesa conservadoraJogadores que ficam offline frequentemente
farmerFoco em coletaDesenvolvimento pacífico

Passo 5: Iniciar​

# Iniciar (estratégia equilibrada padrão)
python -m dark_forest.cli run

# Especificar uma estratégia
python -m dark_forest.cli run --strategy aggressive

Passo 6: Configurar Notificações (Opcional)​

Telegram​

python -m dark_forest.cli config set notify.telegram.token SEU_TOKEN_DO_BOT
python -m dark_forest.cli config set notify.telegram.chat_id SEU_CHAT_ID

Discord​

python -m dark_forest.cli config set notify.discord.webhook SUA_URL_DE_WEBHOOK

Lembretes de Segurança​

  1. 🔐 Use uma carteira de jogo dedicada com limites razoáveis de reserva de DFT/energia
  2. 📉 Abra as permissões gradualmente — comece apenas com COLLECT + REPAIR, depois adicione ATTACK quando tiver certeza de que está estável
  3. 🔄 Verifique regularmente — consulte os logs e relatórios do agente
