Modelos de Estratégia

As estratégias do agente são definidas usando arquivos de configuração YAML.

Estrutura do Arquivo de Estratégia​

agent:
  name: "meu-agente"
  mode: "auto"          # auto | manual | dry-run

limits:                 # Disjuntor (política on-chain)
  min_energy_reserve: 100000
  min_dft_reserve: 10
  max_attacks_per_hour: 5
  max_upgrades_per_day: 3
  max_single_upgrade_cost: 100

combat:                 # Estratégia de combate
  min_target_level: 1
  max_target_level: 999
  weapon_level: 1
  only_attack_when_energy_above: 100000
  blacklist: []         # Endereços na lista negra
  whitelist: []         # Endereços na lista branca (atacar apenas estes)

defense:                # Estratégia de defesa
  auto_repair: true
  min_durability_before_repair: 20
  shield_regen_threshold: 50   # Iniciar regeneração do escudo abaixo deste %

collection:             # Estratégia de coleta
  enabled: true
  interval_minutes: 60

Quatro Estratégias Predefinidas​

1. Agressiva​

combat:
  max_attacks_per_hour: 10
  min_target_level: 0.5
  weapon_level: max

collection:
  enabled: false  # Sem coleta, combate puro
  • ✅ Bom para: jogadores muito ativos que querem recompensas de combate
  • ❌ Risco: alto consumo de energia, risco de ser contra-atacado

2. Equilibrada​

combat:
  max_attacks_per_hour: 5
  min_target_level: 0.8
  weapon_level: balanced

collection:
  enabled: true
  interval_minutes: 30

defense:
  auto_repair: true
  shield_regen_threshold: 60
  • ✅ Recomendada: ataque e defesa equilibrados, adequada para a maioria dos jogadores

3. Defensiva​

combat:
  max_attacks_per_hour: 1
  only_counterattack: true  # Apenas contra-atacar

defense:
  auto_repair: true
  min_durability_before_repair: 50
  shield_regen_threshold: 80

collection:
  enabled: true
  interval_minutes: 15
  • ✅ Bom para: jogadores que ficam offline por longos períodos e querem evitar riscos

4. Fazendeira​

combat:
  enabled: false  # Sem combate

defense:
  auto_repair: true
  shield_regen_threshold: 90

collection:
  enabled: true
  interval_minutes: 5
  • ✅ Bom para: desenvolvimento em estágio inicial e acumulação de recursos

Estratégia Personalizada​

# Copiar o modelo
cp templates/balanced.yaml minha-estrategia.yaml

# Editar parâmetros
vim minha-estrategia.yaml

# Usar estratégia personalizada
python -m dark_forest.cli run --strategy ./minha-estrategia.yaml

Parâmetros do Disjuntor​

ParâmetroDescriçãoValor Padrão
max_attacks_per_hourMáx. ataques por hora10
min_energy_reserveReserva mínima de energia100.000
min_dft_reserveReserva mínima de DFT10
max_upgrades_per_dayMáx. upgrades por dia5

⚠️ Os parâmetros do disjuntor são aplicados on-chain. O agente não pode ignorá-los.
