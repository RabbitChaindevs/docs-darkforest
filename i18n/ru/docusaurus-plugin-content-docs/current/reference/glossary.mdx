# Глоссарий

| Термин | Описание |
|--------|----------|
| **Цивилизация** | Игровая идентичность игрока с собственными уровнями, атрибутами и координатами |
| **Энергия** | Базовый внутриигровой ресурс, используемый для улучшений, атак, передвижения и ремонта |
| **DFT** | Dark Forest Token, токен ERC-20, используемый для прыжков, улучшений, штрафов за выход из альянса и т.д. |
| **Координаты** | Позиция цивилизации в 3D-пространстве `(x, y, z)`, назначаемая по детерминированному алгоритму |
| **Общий уровень** | Сумма уровней всех систем; влияет на стоимость ежедневного обслуживания |
| **Радиус сканирования** | Ваш радиус обнаружения: `1000 + 150×ур + 5×ур²`; определяет, кого вы можете атаковать или преследовать |
| **Сила атаки** | `900 + 10 × ур²`; основной атрибут системы оружия |
| **Защита** | `540 + 6 × ур²`; основной атрибут системы щита |
| **HP щита** | `3600 + 15 × ур²`; поглощает входящий урон |
| **Скорость двигателя** | `10 + 5 × ур`; скорость круиза |
| **Скорость сбора** | `3 + 10 × √(ур - 1)` энергии/секунда |
| **Прочность** | Рабочее состояние каждой системы, расходуется со временем или при использовании |
| **Ежедневное обслуживание** | `2 000 × общий уровень / день`; пропорционально списывается со временем |
| **Вступительный взнос** | BNB, уплачиваемый при создании цивилизации, линейно растёт от 0,01 до 0,05 BNB |
| **Формула якоря** | Формула расчёта стоимости улучшений на основе дневной выработки и якорных дней |
| **Якорные дни** | Уровни 1–100: 1–30 дней; 100–1000: 30–60 дней; 1000+: 60 дней (потолок) |
| **Круиз** | Стандартный метод передвижения — движение к целевым координатам с постоянной скоростью |
| **Космический прыжок** | Мгновенная телепортация к новым координатам, стоимость энергия + DFT, щит падает до 10% |
| **Прыжок-преследование** | Мгновенный прыжок к местоположению цели; требует Радар Ур20+ и цель в радиусе |
| **Токен атаки** | Алгоритм «ведро токенов», ограничивающий частоту атак; макс 3–10 токенов |
| **Коэффициент грабежа** | `5% + ур оружия × 0,5%`; доля энергии цели, изымаемая при победе |
| **Защита нового игрока** | Невозможность быть атакованным в первые 24 часа после создания |
| **Руины** | Цивилизация, которую нельзя далее понизить после поражения; может быть возрождена через Восстановление |
| **Восстановление / Возрождение** | Возрождает цивилизацию в руинах, стоит энергии + DFT, даёт боевой бонус |
| **Боевой бонус** | +12% за каждое восстановление, макс 300% (30000 б.п.) |
| **Альянс** | Организация игроков для сотрудничества, макс 100 участников |
| **Период охлаждения при выходе** | 24-часовой период взаимного ненападения после выхода из альянса |
| **Тотем** | Система баффа для всего альянса; жертвуйте энергию для улучшения и получения бонусов к защите |
| **Эпоха** | Единица времени DailyMinter, от полуночи UTC до следующей полуночи UTC |
| **DELEGATECALL** | Архитектурный шаблон контракта, где основной контракт делегирует логику спутниковым контрактам |
| **Мягкий шлюз** | Когда условия атаки не выполнены, игра показывает причину без расходования ресурсов |
