# قوالب الاستراتيجيات

يتم تعريف استراتيجيات الوكيل باستخدام ملفات تكوين YAML.

---

## هيكل ملف الاستراتيجية

```yaml
agent:
  name: "my-agent"
  mode: "auto"          # auto | manual | dry-run

limits:                 # قاطع الدائرة (سياسة على السلسلة)
  min_energy_reserve: 100000
  min_dft_reserve: 10
  max_attacks_per_hour: 5
  max_upgrades_per_day: 3
  max_single_upgrade_cost: 100

combat:                 # استراتيجية القتال
  min_target_level: 1
  max_target_level: 999
  weapon_level: 1
  only_attack_when_energy_above: 100000
  blacklist: []         # عناوين القائمة السوداء
  whitelist: []         # عناوين القائمة البيضاء (مهاجمة هذه فقط)

defense:                # استراتيجية الدفاع
  auto_repair: true
  min_durability_before_repair: 20
  shield_regen_threshold: 50   # بدء تجديد الدرع عند أقل من هذه %

collection:             # استراتيجية الجمع
  enabled: true
  interval_minutes: 60
```

## الاستراتيجيات الأربع المسبقة

### 1. الهجومية (Aggressive)

```yaml
combat:
  max_attacks_per_hour: 10
  min_target_level: 0.5
  weapon_level: max

collection:
  enabled: false  # لا جمع، قتال خالص
```

- ✅ مناسبة لـ: اللاعبون النشطون جداً الذين يريدون مكافآت قتالية
- ❌ المخاطرة: استهلاك طاقة عالي، خطر الهجوم المضاد

### 2. المتوازنة (Balanced)

```yaml
combat:
  max_attacks_per_hour: 5
  min_target_level: 0.8
  weapon_level: balanced

collection:
  enabled: true
  interval_minutes: 30

defense:
  auto_repair: true
  shield_regen_threshold: 60
```

- ✅ **موصى بها**: هجوم ودفاع متوازنان، مناسبة لمعظم اللاعبين

### 3. الدفاعية (Defensive)

```yaml
combat:
  max_attacks_per_hour: 1
  only_counterattack: true  # هجوم مضاد فقط

defense:
  auto_repair: true
  min_durability_before_repair: 50
  shield_regen_threshold: 80

collection:
  enabled: true
  interval_minutes: 15
```

- ✅ مناسبة لـ: اللاعبون غير المتصلين لفترات طويلة ويريدون تجنب المخاطر

### 4. الجامعة (Farmer)

```yaml
combat:
  enabled: false  # لا قتال

defense:
  auto_repair: true
  shield_regen_threshold: 90

collection:
  enabled: true
  interval_minutes: 5
```

- ✅ مناسبة لـ: التطوير في المرحلة المبكرة وتجميع الموارد

## استراتيجية مخصصة

```bash
# نسخ القالب
cp templates/balanced.yaml my-strategy.yaml

# تعديل المعاملات
vim my-strategy.yaml

# استخدام الاستراتيجية المخصصة
python -m dark_forest.cli run --strategy ./my-strategy.yaml
```

---

## معاملات قاطع الدائرة

| المعامل | الوصف | القيمة الافتراضية |
|-----------|-------------|---------|
| `max_attacks_per_hour` | الحد الأقصى للهجمات في الساعة | 10 |
| `min_energy_reserve` | الحد الأدنى لاحتياطي الطاقة | 100,000 |
| `min_dft_reserve` | الحد الأدنى لاحتياطي DFT | 10 |
| `max_upgrades_per_day` | الحد الأقصى للترقيات في اليوم | 5 |

> ⚠️ يتم تطبيق معاملات قاطع الدائرة على السلسلة. لا يمكن للوكيل تجاوزها.

**الخطوة التالية الموصى بها: [عناوين العقود →](/docs/reference/addresses)**
