# AI エージェントとは

AI エージェントは、あなたの文明に代わってゲーム操作を実行する **オフチェーン自動プログラム** です。**権限ビットマスク** によって認可され、**ポリシー制約** の範囲内で動作します。

---

## エージェントができること

エージェントの機能は **認可された権限** によって制御されます：

| 権限 | 値 | アクション |
|-----------|-------|--------|
| COLLECT | 1 | エネルギー収集 |
| REPAIR | 2 | システム修理 |
| CLAIM_COMBAT | 8 | 戦闘エネルギーの請求 |
| ATTACK | 32 | 攻撃 |
| UPGRADE | 64 | システムアップグレード |
| JUMP | 128 | 宇宙ジャンプ |
| REBUILD | 256 | 文明の再建 |
| DONATE | 512 | トーテムへの寄付 |

任意の権限の組み合わせをエージェントに付与できます（ビットマスク）。

---

## ポリシー制約

権限に加えて、エージェントに **ポリシー制約** を設定できます：

| 制約 | 説明 |
|-----------|-------------|
| `minEnergyReserve` | 最小エネルギー準備金（これを下回るとエージェントは消費しない） |
| `minDftReserve` | 最小 DFT 準備金 |
| `maxAttackPerHour` | 1時間あたりの最大攻撃回数 |
| `maxUpgradePerDay` | 1日あたりの最大アップグレード回数 |
| `maxSingleUpgradeCost` | アップグレードあたりの最大 DFT コスト |
| `autoCollect` | 自動エネルギー収集 |
| `autoRepair` | 自動システム修理 |
| `autoShieldRepair` | 自動シールド修理 |
| `autoUpgrade` | 自動アップグレード |
| `autoAttack` | 自動攻撃 |

## セキュリティメカニズム

### 1. オンチェーン登録

エージェントのポリシーは `DarkForestAgentRegistry.sol` コントラクトに保存され、一度設定すると改ざんできません。

### 2. 権限の分離

- **認可**：エージェントがあなたの代わりに行動することを認可
- **権限ビットマスク**：各操作を細かく制御
- **取り消し可能**：`revokeAgent()` は即座に有効

### 3. サーキットブレーカー

```
攻撃頻度制限：最大 N 回/時間
アップグレード頻度制限：最大 N 回/日
エネルギーフロア：X を下回らない
DFT フロア：Y を下回らない
```

### 4. ブラックリスト / ホワイトリスト

- **ブラックリスト**：特定のプレイヤーへの攻撃を禁止
- **ホワイトリスト**：特定のプレイヤーのみ攻撃を許可（ホワイトリスト有効時はブラックリストは無視）

## エージェント vs 手動プレイ

| 比較 | 手動 | AI エージェント |
|-----------|--------|----------|
| オンライン要件 | 頻繁なオンラインが必要 | 24時間365日稼働 |
| 応答時間 | レイテンシあり | 即時 |
| 精度 | エラーが発生しやすい | ポリシーに厳密に従う |
| セキュリティ | ウォレットに秘密鍵 | 暗号化ストレージ + オンチェーン制約 |
