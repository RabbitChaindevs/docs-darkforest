# エネルギーマーケット

エネルギーマーケットは **分散型オーダーブック** であり、プレイヤーは DFT でエネルギーを取引できます。`EnergyMarket.sol` コントラクトによって動作します。

---

## エネルギーの売却（注文作成）

### 操作

```
createOrder(energyAmount, dftPrice)
```

1. 売却したい **エネルギー量** を入力
2. 希望する **DFT での合計価格** を入力
3. トランザクションを確認

### エネルギーのロック

注文を作成すると、コントラクトは即座にエネルギーをロックし（`energyReserved` とマーク）、注文がアクティブな間は使用できなくなります。

### 注文のキャンセル

```
cancelOrder(orderId)
```

- キャンセルには注文作成から少なくとも **3秒** 経過している必要あり（同一ブロック内でのフロントランニングを防止）
- キャンセル後、エネルギーはロック解除される

---

## エネルギーの購入（注文の約定）

### 操作

```
fillOrder(orderId, maxPrice)
```

1. アクティブな売り注文一覧を参照
2. 注文を選択
3. 受け入れ可能な最大価格を入力
4. トランザクションを確認

### 手数料

売り手の受け取り額から **1% の DFT プロトコル手数料** が差し引かれます：

```
買い手の支払額 = 注文価格（表示価格そのまま）
売り手の受取額 = 注文価格 × 99%
プロトコル手数料 = 注文価格 × 1%（コントラクトに蓄積）
```

1% のプロトコル手数料はコントラクトに蓄積され、管理者が引き出すことができます。

---

## マーケットパラメータ

| パラメータ | 値 |
|-----------|-------|
| プロトコル手数料 | 1%（売り手から控除） |
| 最小注文経過時間 | 3秒（キャンセル/約定前） |
| 注文ステータス | active / inactive（キャンセルまたは約定後に inactive に） |

---

## 取引例

### Alice がエネルギーを売却

1. Alice が売り注文を作成：`createOrder(10000, 100 DFT)` — 10,000 エネルギーを合計100 DFTで売却
2. コントラクトが Alice の10,000 エネルギーをロック

### Bob がエネルギーを購入

1. Bob が Alice の注文を確認
2. `fillOrder(orderId, 100)` を呼び出し — ちょうど100 DFTを支払う
3. Alice は99 DFTを受け取る（100 - 1% 手数料）
4. 1 DFT がプロトコル手数料として蓄積

---

## 戦略ヒント

| シナリオ | 提案 |
|----------|------------|
| 戦闘用に緊急でエネルギーが必要 | 最安値の売り注文を探す |
| 急いでいない | 指値注文を出して適切な価格を待つ |
| DFT価格が上昇中 | エネルギーをDFTに交換して利益を得る |
| エネルギーが余っている | 売り注文を出してDFTを稼ぐ |
