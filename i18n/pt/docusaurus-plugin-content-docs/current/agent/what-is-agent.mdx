# O que é um AI Agent?

Um AI Agent é um **programa automatizado off-chain** que realiza operações do jogo em nome da sua civilização. Ele é autorizado através de uma **máscara de permissões** e opera dentro de **restrições de políticas**.

---

## O que um Agent Pode Fazer

As capacidades do agente são controladas por **permissões autorizadas**:

| Permissão | Valor | Ação |
|-----------|-------|------|
| COLLECT | 1 | Coletar energia |
| REPAIR | 2 | Reparar sistemas |
| CLAIM_COMBAT | 8 | Reivindicar energia de combate |
| ATTACK | 32 | Atacar |
| UPGRADE | 64 | Melhorar sistemas |
| JUMP | 128 | Salto espacial |
| REBUILD | 256 | Reconstruir civilização |
| DONATE | 512 | Doar ao totem |

Você pode conceder qualquer combinação de permissões a um agente (máscara de bits).

---

## Restrições de Políticas

Além das permissões, você pode definir **restrições de políticas** para o agente:

| Restrição | Descrição |
|-----------|-----------|
| `minEnergyReserve` | Reserva mínima de energia (o agente não gastará abaixo disso) |
| `minDftReserve` | Reserva mínima de DFT |
| `maxAttackPerHour` | Máximo de ataques por hora |
| `maxUpgradePerDay` | Máximo de upgrades por dia |
| `maxSingleUpgradeCost` | Custo máximo de DFT por upgrade |
| `autoCollect` | Coletar energia automaticamente |
| `autoRepair` | Reparar sistemas automaticamente |
| `autoShieldRepair` | Reparar escudo automaticamente |
| `autoUpgrade` | Melhorar automaticamente |
| `autoAttack` | Atacar automaticamente |

## Mecanismos de Segurança

### 1. Registro On-Chain

A política do agente é armazenada no contrato `DarkForestAgentRegistry.sol` e não pode ser adulterada após ser definida.

### 2. Separação de Permissões

- **Autorizar**: Você autoriza o agente a agir em seu nome
- **Máscara de permissões**: Controle granular sobre cada operação
- **Revogável**: `revokeAgent()` tem efeito imediato

### 3. Disjuntores

```
Limite de frequência de ataque: máx. N por hora
Limite de frequência de upgrade: máx. N por dia
Piso de energia: nunca abaixo de X
Piso de DFT: nunca abaixo de Y
```

### 4. Lista Negra / Lista Branca

- **Lista negra**: Proibir atacar jogadores específicos
- **Lista branca**: Permitir atacar apenas jogadores específicos (a lista negra é ignorada quando a lista branca está ativa)

## Arquitetura de Duas Camadas

```
┌─────────────────────────────────────┐
│         Agente Off-Chain            │
│  ┌──────────────────────────────┐   │
│  │ Motor de Estratégia (YAML)   │   │
│  │ 4 estratégias predefinidas   │   │
│  │ Modelos de estratégia        │   │
│  │ personalizados               │   │
│  └──────────┬───────────────────┘   │
│             │ via chamadas a        │
│             │ contratos             │
│             ▼                      │
│  ┌──────────────────────────────┐   │
│  │ Agent SDK                    │   │
│  │ Chama métodos do contrato    │   │
│  │ (leitura/escrita)            │   │
│  └──────────────────────────────┘   │
└──────────────┬──────────────────────┘
               │
               ▼
┌─────────────────────────────────────┐
│   On-Chain DarkForestAgentRegistry  │
│  ┌──────────────────────────────┐   │
│  │ Verificação de máscara de    │   │
│  │ permissões                   │   │
│  │ Verificação de restrições    │   │
│  │ de políticas                 │   │
│  │ Verificação de limite de     │   │
│  │ frequência                   │   │
│  └──────────────────────────────┘   │
└─────────────────────────────────────┘
```

## Agent vs Jogo Manual

| Comparação | Manual | AI Agent |
|------------|--------|----------|
| Requisito de conexão | Precisa estar online frequentemente | Operação 24/7 |
| Tempo de resposta | Latência | Instantâneo |
| Precisão | Propenso a erros | Segue estritamente a política |
| Segurança | Chave privada na carteira | Armazenamento criptografado + restrições on-chain |
