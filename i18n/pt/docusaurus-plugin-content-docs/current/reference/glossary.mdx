# Glossário

| Termo | Descrição |
|-------|-----------|
| **Civilização** | A identidade do jogador no jogo com seus próprios níveis, atributos e coordenadas |
| **Energia** | O recurso básico do jogo usado para upgrades, ataques, movimento e reparos |
| **DFT** | Dark Forest Token, um token ERC-20 usado para saltos, upgrades, multas de saída de aliança e mais |
| **Coordenadas** | A posição de uma civilização no espaço 3D `(x, y, z)`, atribuída via algoritmo determinístico |
| **Nível Total** | A soma de todos os níveis do sistema; afeta os custos de manutenção diária |
| **Alcance de Escaneamento** | Seu raio de detecção: `1000 + 150×nv + 5×nv²`; determina quem você pode atacar ou rastrear |
| **Poder de Ataque** | `900 + 10 × nível²`; o atributo principal do sistema de Armas |
| **Defesa** | `540 + 6 × nível²`; o atributo principal do sistema de Escudo |
| **HP do Escudo** | `3600 + 15 × nível²`; absorve o dano recebido |
| **Velocidade do Motor** | `10 + 5 × nível`; velocidade de cruzeiro |
| **Taxa de Coleta** | `3 + 10 × √(nível - 1)` energia/segundo |
| **Durabilidade** | O estado de funcionamento de cada sistema, consumido com o tempo ou através do uso |
| **Manutenção Diária** | `2.000 × nível total / dia`; deduzido proporcionalmente ao longo do tempo |
| **Taxa de Entrada** | BNB pago ao criar uma civilização, com crescimento linear de 0,01 a 0,05 BNB |
| **Fórmula Âncora** | A fórmula para calcular custos de upgrade baseada na produção diária e dias âncora |
| **Dias Âncora** | Nível 1–100: 1–30 dias; 100–1000: 30–60 dias; 1000+: 60 dias (teto) |
| **Cruzeiro** | Método de movimento padrão — move-se em direção às coordenadas alvo em velocidade constante |
| **Salto Espacial** | Teletransporte instantâneo para novas coordenadas, custa energia + DFT, escudo cai para 10% |
| **Salto de Rastreamento** | Salto instantâneo para a localização de um alvo; requer Radar Nv20+ e alvo dentro do alcance |
| **Token de Ataque** | Algoritmo de balde de tokens que limita a frequência de ataque; máx. 3–10 tokens |
| **Taxa de Saque** | `5% + nível de arma × 0,5%`; porção da energia do alvo saqueada ao vencer |
| **Proteção de Novos Jogadores** | Não pode ser atacado nas primeiras 24 horas após a criação |
| **Ruínas** | Uma civilização que não pode ser mais rebaixada após a derrota; pode ser revivida via Reconstruir |
| **Reconstruir / Renascimento** | Revive uma civilização em ruínas, custa energia + DFT, concede bônus de combate |
| **Bônus de Combate** | +12% por reconstrução, máx. 300% (30000 bps) |
| **Aliança** | Uma organização de jogadores para colaboração, máx. 100 membros |
| **Resfriamento de Saída** | Período de não agressão mútua de 24 horas após sair de uma aliança |
| **Totem** | Um sistema de melhoria para toda a aliança; doe energia para melhorá-lo e obter bônus defensivos |
| **Época** | Unidade de tempo do DailyMinter, da meia-noite UTC à próxima meia-noite UTC |
| **DELEGATECALL** | Padrão de arquitetura de contratos onde o contrato principal delega lógica a contratos satélite |
| **Porta Suave** | Quando as condições de ataque não são atendidas, o jogo mostra o motivo sem consumir recursos |
