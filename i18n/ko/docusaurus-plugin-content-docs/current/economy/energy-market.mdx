# 에너지 시장

에너지 시장은 **탈중앙화 오더북**으로, 플레이어가 에너지를 DFT로 거래할 수 있습니다. `EnergyMarket.sol` 컨트랙트로 구동됩니다.

---

## 에너지 판매 (주문 생성)

### 작동

```
createOrder(energyAmount, dftPrice)
```

1. 판매할 **에너지 수량**을 입력합니다
2. 요구하는 **DFT 총 가격**을 입력합니다
3. 트랜잭션을 확인합니다

### 에너지 잠금

주문을 생성하면 컨트랙트가 즉시 에너지를 **잠급니다** (`energyReserved`로 표시). 주문이 활성 상태인 동안 에너지를 사용할 수 없습니다.

### 주문 취소

```
cancelOrder(orderId)
```

- 주문은 취소 전 **최소 3초**가 경과해야 합니다 (동일 블록 프론트러닝 방지)
- 취소 후 에너지 잠금이 해제됩니다

---

## 에너지 구매 (주문 체결)

### 작동

```
fillOrder(orderId, maxPrice)
```

1. 활성 판매 주문 목록을 둘러봅니다
2. 주문을 선택합니다
3. 수락할 최대 가격을 입력합니다
4. 트랜잭션을 확인합니다

### 수수료

판매자 수익에서 **1% DFT 프로토콜 수수료**가 차감됩니다:

```
구매자 지불 = 주문 가격 (정확한 리스팅 가격)
판매자 수령 = 주문 가격 × 99%
프로토콜 수수료 = 주문 가격 × 1% (컨트랙트에 누적)
```

1% 프로토콜 수수료는 컨트랙트에 누적되며 관리자가 인출할 수 있습니다.

---

## 시장 파라미터

| 파라미터 | 값 |
|---------|-----|
| 프로토콜 수수료 | 1% (판매자에게 차감) |
| 최소 주문 유지 시간 | 3초 (취소/체결 전) |
| 주문 상태 | 활성 / 비활성 (취소 또는 체결 후 비활성화) |

---

## 거래 예시

### Alice가 에너지 판매

1. Alice가 판매 주문 생성: `createOrder(10000, 100 DFT)` — 10,000 에너지를 총 100 DFT에 판매
2. 컨트랙트가 Alice의 10,000 에너지를 잠금

### Bob이 에너지 구매

1. Bob이 Alice의 주문을 확인
2. `fillOrder(orderId, 100)` 호출 — 정확히 100 DFT 지불
3. Alice는 99 DFT 수령 (100 - 1% 수수료)
4. 1 DFT가 프로토콜 수수료로 누적

---

## 전략 팁

| 상황 | 제안 |
|------|------|
| 전투를 위해 에너지가 긴급히 필요할 때 | 가장 낮은 가격의 판매 주문 수락 |
| 급하지 않을 때 | 지정가 주문을 걸고 적절한 가격을 기다림 |
| DFT 가격이 상승 중일 때 | 에너지를 DFT로 판매하여 수익 실현 |
| 에너지가 남아돌 때 | 판매 주문을 등록하여 DFT 획득 |
