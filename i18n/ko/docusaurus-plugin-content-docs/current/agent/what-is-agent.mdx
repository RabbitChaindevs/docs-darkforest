# AI 에이전트란?

AI 에이전트는 사용자의 문명을 대신하여 게임 작업을 수행하는 **오프체인 자동화 프로그램**입니다. **권한 비트마스크**를 통해 승인되며 **정책 제약 조건** 내에서 작동합니다.

---

## 에이전트가 할 수 있는 일

에이전트의 기능은 **승인된 권한**에 의해 제어됩니다:

| 권한 | 값 | 액션 |
|------|-----|------|
| COLLECT | 1 | 에너지 수집 |
| REPAIR | 2 | 시스템 수리 |
| CLAIM_COMBAT | 8 | 전투 에너지 청구 |
| ATTACK | 32 | 공격 |
| UPGRADE | 64 | 시스템 업그레이드 |
| JUMP | 128 | 스페이스 점프 |
| REBUILD | 256 | 문명 재건 |
| DONATE | 512 | 토템 기부 |

원하는 권한 조합을 에이전트에 부여할 수 있습니다 (비트마스크).

---

## 정책 제약 조건

권한 외에도 에이전트에 **정책 제약 조건**을 설정할 수 있습니다:

| 제약 조건 | 설명 |
|-----------|------|
| `minEnergyReserve` | 최소 에너지 보유량 (이 이하로는 사용 불가) |
| `minDftReserve` | 최소 DFT 보유량 |
| `maxAttackPerHour` | 시간당 최대 공격 횟수 |
| `maxUpgradePerDay` | 일일 최대 업그레이드 횟수 |
| `maxSingleUpgradeCost` | 업그레이드당 최대 DFT 비용 |
| `autoCollect` | 자동 에너지 수집 |
| `autoRepair` | 자동 시스템 수리 |
| `autoShieldRepair` | 자동 방어막 수리 |
| `autoUpgrade` | 자동 업그레이드 |
| `autoAttack` | 자동 공격 |

## 보안 메커니즘

### 1. 온체인 등록

에이전트의 정책은 `DarkForestAgentRegistry.sol` 컨트랙트에 저장되며, 한 번 설정되면 변조할 수 없습니다.

### 2. 권한 분리

- **승인**: 에이전트가 사용자를 대신해 행동하도록 승인
- **권한 비트마스크**: 각 작업에 대한 세분화된 제어
- **철회 가능**: `revokeAgent()`가 즉시 적용됨

### 3. 서킷 브레이커

```
공격 빈도 제한: 시간당 최대 N회
업그레이드 빈도 제한: 일일 최대 N회
에너지 최소치: X 미만으로 떨어지지 않음
DFT 최소치: Y 미만으로 떨어지지 않음
```

### 4. 블랙리스트 / 화이트리스트

- **블랙리스트**: 특정 플레이어 공격 금지
- **화이트리스트**: 특정 플레이어만 공격 허용 (화이트리스트 활성 시 블랙리스트 무시)

## 에이전트 vs 수동 플레이

| 비교 | 수동 | AI 에이전트 |
|------|------|-----------|
| 온라인 요구사항 | 자주 접속 필요 | 24/7 운영 |
| 응답 시간 | 지연 시간 | 즉시 |
| 정밀도 | 오류 가능성 | 정책을 엄격히 준수 |
| 보안 | 지갑에 개인키 보관 | 암호화 저장 + 온체인 제약 |
