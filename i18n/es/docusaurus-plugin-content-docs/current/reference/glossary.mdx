# Glosario

| Término | Descripción |
|---------|-------------|
| **Civilización** | La identidad del jugador en el juego con sus propios niveles, atributos y coordenadas |
| **Energía** | El recurso básico del juego utilizado para mejoras, ataques, movimiento y reparaciones |
| **DFT** | Dark Forest Token, un token ERC-20 utilizado para saltos, mejoras, multas de salida de alianza y más |
| **Coordenadas** | La posición de una civilización en el espacio 3D `(x, y, z)`, asignada mediante un algoritmo determinista |
| **Nivel Total** | La suma de todos los niveles del sistema; afecta los costos de mantenimiento diario |
| **Rango de Escaneo** | Tu radio de detección: `1000 + 150×lv + 5×lv²`; determina a quién puedes atacar o rastrear |
| **Poder de Ataque** | `900 + 10 × nivel²`; el atributo principal del sistema de Armas |
| **Defensa** | `540 + 6 × nivel²`; el atributo principal del sistema de Escudo |
| **HP del Escudo** | `3600 + 15 × nivel²`; absorbe el daño entrante |
| **Velocidad del Motor** | `10 + 5 × nivel`; velocidad de crucero |
| **Tasa de Recolección** | `3 + 10 × √(nivel - 1)` energía/segundo |
| **Durabilidad** | El estado de funcionamiento de cada sistema, consumido con el tiempo o mediante el uso |
| **Mantenimiento Diario** | `2.000 × nivel total / día`; deducido proporcionalmente con el tiempo |
| **Tarifa de Entrada** | BNB pagado al crear una civilización, con crecimiento lineal de 0,01 a 0,05 BNB |
| **Fórmula Ancla** | La fórmula para calcular los costos de mejora basada en la producción diaria y los días ancla |
| **Días Ancla** | Nivel 1–100: 1–30 días; 100–1000: 30–60 días; 1000+: 60 días (tope) |
| **Crucero** | Método de movimiento estándar — se mueve hacia las coordenadas objetivo a velocidad constante |
| **Salto Espacial** | Teletransporte instantáneo a nuevas coordenadas, cuesta energía + DFT, el escudo baja al 10% |
| **Salto de Seguimiento** | Salto instantáneo a la ubicación de un objetivo; requiere Radar Nv20+ y objetivo en alcance |
| **Token de Ataque** | Algoritmo de cubeta de tokens que limita la frecuencia de ataque; máx. 3–10 tokens |
| **Ratio de Saqueo** | `5% + nivel de arma × 0,5%`; porción de la energía del objetivo saqueada al vencer |
| **Protección de Nuevos Jugadores** | No se puede ser atacado durante las primeras 24 horas después de la creación |
| **Ruinas** | Una civilización que no puede ser degradada más después de la derrota; puede ser revivida mediante Reconstruir |
| **Reconstruir / Renacimiento** | Revive una civilización en ruinas, cuesta energía + DFT, otorga mejora de combate |
| **Mejora de Combate** | +12% por reconstrucción, máx. 300% (30000 bps) |
| **Alianza** | Una organización de jugadores para la colaboración, máx. 100 miembros |
| **Enfriamiento de Salida** | Período de no agresión mutua de 24 horas después de abandonar una alianza |
| **Tótem** | Un sistema de mejora para toda la alianza; dona energía para mejorarlo y obtener bonificaciones defensivas |
| **Época** | Unidad de tiempo del DailyMinter, desde la medianoche UTC hasta la siguiente medianoche UTC |
| **DELEGATECALL** | Patrón de arquitectura de contratos donde el contrato principal delega la lógica a contratos satélite |
| **Puerta Blanda** | Cuando no se cumplen las condiciones de ataque, el juego muestra la razón sin consumir recursos |
