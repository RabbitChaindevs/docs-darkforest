# ¿Qué es un AI Agent?

Un AI Agent es un **programa automatizado fuera de cadena** que realiza operaciones del juego en nombre de tu civilización. Está autorizado a través de una **máscara de permisos** y opera dentro de **restricciones de políticas**.

---

## Lo que un Agent Puede Hacer

Las capacidades del agente están controladas por **permisos autorizados**:

| Permiso | Valor | Acción |
|---------|-------|--------|
| COLLECT | 1 | Recolectar energía |
| REPAIR | 2 | Reparar sistemas |
| CLAIM_COMBAT | 8 | Reclamar energía de combate |
| ATTACK | 32 | Atacar |
| UPGRADE | 64 | Mejorar sistemas |
| JUMP | 128 | Salto espacial |
| REBUILD | 256 | Reconstruir civilización |
| DONATE | 512 | Donar al tótem |

Puedes conceder cualquier combinación de permisos a un agente (máscara de bits).

---

## Restricciones de Políticas

Además de los permisos, puedes establecer **restricciones de políticas** para el agente:

| Restricción | Descripción |
|-------------|-------------|
| `minEnergyReserve` | Reserva mínima de energía (el agente no gastará por debajo de esto) |
| `minDftReserve` | Reserva mínima de DFT |
| `maxAttackPerHour` | Máximo de ataques por hora |
| `maxUpgradePerDay` | Máximo de mejoras por día |
| `maxSingleUpgradeCost` | Costo máximo de DFT por mejora |
| `autoCollect` | Recolectar energía automáticamente |
| `autoRepair` | Reparar sistemas automáticamente |
| `autoShieldRepair` | Reparar escudo automáticamente |
| `autoUpgrade` | Mejorar automáticamente |
| `autoAttack` | Atacar automáticamente |

## Mecanismos de Seguridad

### 1. Registro en Cadena

La política del agente se almacena en el contrato `DarkForestAgentRegistry.sol` y no puede ser manipulada una vez establecida.

### 2. Separación de Permisos

- **Autorizar**: Autorizas al agente para actuar en tu nombre
- **Máscara de permisos**: Control granular sobre cada operación
- **Revocable**: `revokeAgent()` tiene efecto inmediato

### 3. Interruptores de Seguridad

```
Límite de frecuencia de ataque: máx. N por hora
Límite de frecuencia de mejora: máx. N por día
Límite mínimo de energía: nunca por debajo de X
Límite mínimo de DFT: nunca por debajo de Y
```

### 4. Lista Negra / Lista Blanca

- **Lista negra**: Prohibir atacar a jugadores específicos
- **Lista blanca**: Solo permitir atacar a jugadores específicos (la lista negra se ignora cuando la lista blanca está activa)

## Agent vs Juego Manual

| Comparación | Manual | AI Agent |
|-------------|--------|----------|
| Requisito de conexión | Debe estar conectado a menudo | Operación 24/7 |
| Tiempo de respuesta | Latencia | Instantáneo |
| Precisión | Propenso a errores | Sigue estrictamente la política |
| Seguridad | Clave privada en la billetera | Almacenamiento cifrado + restricciones en cadena |
