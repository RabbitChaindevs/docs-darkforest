# शब्दावली

| शब्द | विवरण |
|------|-------------|
| **सभ्यता** | खिलाड़ी की इन-गेम पहचान जिसके अपने स्तर, गुण और निर्देशांक होते हैं |
| **ऊर्जा** | मूल इन-गेम संसाधन, अपग्रेड, हमले, गति और मरम्मत के लिए उपयोग किया जाता है |
| **DFT** | डार्क फ़ॉरेस्ट टोकन, एक ERC-20 टोकन जो जंप, अपग्रेड, गठबंधन निकास जुर्माना और अधिक के लिए उपयोग किया जाता है |
| **निर्देशांक** | 3D अंतरिक्ष में एक सभ्यता की स्थिति `(x, y, z)`, एक नियतात्मक एल्गोरिदम के माध्यम से निर्धारित |
| **कुल स्तर** | सभी सिस्टम स्तरों का योग; दैनिक रखरखाव लागत को प्रभावित करता है |
| **स्कैन रेंज** | आपकी पहचान त्रिज्या: `1000 + 150×lv + 5×lv²`; निर्धारित करता है कि आप किस पर हमला या ट्रैक कर सकते हैं |
| **हमला शक्ति** | `900 + 10 × level²`; हथियार सिस्टम का मुख्य गुण |
| **रक्षा** | `540 + 6 × level²`; ढाल सिस्टम का मुख्य गुण |
| **ढाल HP** | `3600 + 15 × level²`; आने वाली क्षति को अवशोषित करता है |
| **इंजन गति** | `10 + 5 × level`; क्रूज़ गति |
| **संग्रह दर** | `3 + 10 × √(level - 1)` ऊर्जा/सेकंड |
| **स्थायित्व** | प्रत्येक सिस्टम की कार्यशील स्थिति, समय या उपयोग के साथ खपत |
| **दैनिक रखरखाव** | `2,000 × total level / day`; समयानुपातिक रूप से काटा जाता है |
| **प्रवेश शुल्क** | सभ्यता बनाते समय भुगतान किया गया BNB, 0.01 से 0.05 BNB रैखिक वृद्धि |
| **एंकर फ़ॉर्मूला** | दैनिक उत्पादन और एंकर दिनों के आधार पर अपग्रेड लागत की गणना का फ़ॉर्मूला |
| **एंकर दिन** | स्तर 1–100: 1–30 दिन; 100–1000: 30–60 दिन; 1000+: 60 दिन (सीमित) |
| **क्रूज़** | मानक गति विधि — स्थिर गति से लक्ष्य निर्देशांक की ओर बढ़ता है |
| **स्पेस जंप** | नए निर्देशांक पर तत्काल टेलीपोर्ट, ऊर्जा + DFT खर्च करता है, ढाल 10% पर गिर जाती है |
| **ट्रैकिंग जंप** | लक्ष्य के स्थान पर तत्काल जंप; रडार Lv20+ और लक्ष्य रेंज में आवश्यक |
| **हमला टोकन** | हमला आवृत्ति सीमित करने वाला टोकन बकेट एल्गोरिदम; अधिकतम 3–10 टोकन |
| **लूट अनुपात** | `5% + weapon level × 0.5%`; जीत पर लक्ष्य ऊर्जा का हिस्सा लूटा जाता है |
| **नए खिलाड़ी सुरक्षा** | निर्माण के बाद पहले 24 घंटों तक हमला नहीं किया जा सकता |
| **खंडहर** | एक सभ्यता जो हार के बाद और डाउनग्रेड नहीं की जा सकती; पुनर्निर्माण के माध्यम से पुनर्जीवित की जा सकती है |
| **पुनर्निर्माण / पुनर्जन्म** | खंडहर सभ्यता को पुनर्जीवित करता है, ऊर्जा + DFT खर्च करता है, युद्ध बूस्ट प्रदान करता है |
| **युद्ध बूस्ट** | प्रति पुनर्निर्माण +12%, अधिकतम 300% (30000 bps) |
| **गठबंधन** | सहयोग के लिए खिलाड़ी संगठन, अधिकतम 100 सदस्य |
| **लीव कूलडाउन** | गठबंधन छोड़ने के बाद 24 घंटे की पारस्परिक गैर-आक्रामकता अवधि |
| **टोटम** | एक गठबंधन-व्यापी बफ प्रणाली; रक्षा बोनस के लिए ऊर्जा दान करें और अपग्रेड करें |
| **युग** | DailyMinter की समय इकाई, UTC मध्यरात्रि से अगली UTC मध्यरात्रि तक |
| **DELEGATECALL** | कॉन्ट्रैक्ट आर्किटेक्चर पैटर्न जहाँ मुख्य कॉन्ट्रैक्ट उपग्रह कॉन्ट्रैक्ट को लॉजिक सौंपता है |
| **सॉफ्ट गेट** | जब हमला करने की शर्तें पूरी नहीं होतीं, गेम संसाधन खपत किए बिना कारण दिखाता है |
