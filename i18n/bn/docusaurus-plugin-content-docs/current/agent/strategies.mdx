# কৌশল টেমপ্লেট

এজেন্ট কৌশলগুলি YAML কনফিগারেশন ফাইল ব্যবহার করে সংজ্ঞায়িত করা হয়।

---

## কৌশল ফাইল কাঠামো

```yaml
agent:
  name: "my-agent"
  mode: "auto"          # auto | manual | dry-run

limits:                 # সার্কিট ব্রেকার (অন-চেইন নীতি)
  min_energy_reserve: 100000
  min_dft_reserve: 10
  max_attacks_per_hour: 5
  max_upgrades_per_day: 3
  max_single_upgrade_cost: 100

combat:                 # যুদ্ধ কৌশল
  min_target_level: 1
  max_target_level: 999
  weapon_level: 1
  only_attack_when_energy_above: 100000
  blacklist: []         # ব্ল্যাকলিস্টেড ঠিকানা
  whitelist: []         # হোয়াইটলিস্টেড ঠিকানা (শুধুমাত্র এদের আক্রমণ)

defense:                # প্রতিরক্ষা কৌশল
  auto_repair: true
  min_durability_before_repair: 20
  shield_regen_threshold: 50   # এই %-এর নিচে ঢাল পুনর্জন্ম শুরু

collection:             # সংগ্রহ কৌশল
  enabled: true
  interval_minutes: 60
```

## চারটি প্রিসেট কৌশল

### 1. আক্রমণাত্মক (Aggressive)

```yaml
combat:
  max_attacks_per_hour: 10
  min_target_level: 0.5
  weapon_level: max

collection:
  enabled: false  # কোন সংগ্রহ নয়, সম্পূর্ণ যুদ্ধ
```

- ✅ ভালো: অত্যন্ত সক্রিয় খেলোয়াড়দের জন্য যারা যুদ্ধ পুরস্কার চান
- ❌ ঝুঁকি: উচ্চ শক্তি খরচ, পাল্টা আক্রমণের ঝুঁকি

### 2. ভারসাম্যপূর্ণ (Balanced)

```yaml
combat:
  max_attacks_per_hour: 5
  min_target_level: 0.8
  weapon_level: balanced

collection:
  enabled: true
  interval_minutes: 30

defense:
  auto_repair: true
  shield_regen_threshold: 60
```

- ✅ **প্রস্তাবিত**: ভারসাম্যপূর্ণ আক্রমণ ও প্রতিরক্ষা, অধিকাংশ খেলোয়াড়ের জন্য উপযোগী

### 3. প্রতিরক্ষামূলক (Defensive)

```yaml
combat:
  max_attacks_per_hour: 1
  only_counterattack: true  # শুধুমাত্র পাল্টা আক্রমণ

defense:
  auto_repair: true
  min_durability_before_repair: 50
  shield_regen_threshold: 80

collection:
  enabled: true
  interval_minutes: 15
```

- ✅ ভালো: খেলোয়াড়দের জন্য যারা দীর্ঘ সময় অফলাইনে থাকেন এবং ঝুঁকি এড়াতে চান

### 4. কৃষক (Farmer)

```yaml
combat:
  enabled: false  # কোন যুদ্ধ নয়

defense:
  auto_repair: true
  shield_regen_threshold: 90

collection:
  enabled: true
  interval_minutes: 5
```

- ✅ ভালো: প্রারম্ভিক-গেম উন্নয়ন এবং সম্পদ সঞ্চয়ের জন্য

## কাস্টম কৌশল

```bash
# টেমপ্লেট কপি করুন
cp templates/balanced.yaml my-strategy.yaml

# প্যারামিটার সম্পাদনা করুন
vim my-strategy.yaml

# কাস্টম কৌশল ব্যবহার করুন
python -m dark_forest.cli run --strategy ./my-strategy.yaml
```

---

## সার্কিট ব্রেকার প্যারামিটার

| প্যারামিটার | বিবরণ | ডিফল্ট |
|-----------|-------------|---------|
| `max_attacks_per_hour` | প্রতি ঘন্টায় সর্বোচ্চ আক্রমণ | 10 |
| `min_energy_reserve` | ন্যূনতম শক্তি মজুদ | 100,000 |
| `min_dft_reserve` | ন্যূনতম DFT মজুদ | 10 |
| `max_upgrades_per_day` | প্রতিদিন সর্বোচ্চ আপগ্রেড | 5 |

> ⚠️ সার্কিট ব্রেকার প্যারামিটারগুলি অন-চেইনে প্রয়োগ করা হয়। এজেন্ট সেগুলি বাইপাস করতে পারে না।
