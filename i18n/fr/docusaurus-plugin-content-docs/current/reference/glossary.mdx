# Glossaire

| Terme | Description |
|-------|-------------|
| **Civilisation** | L'identité du joueur dans le jeu avec ses propres niveaux, attributs et coordonnées |
| **Énergie** | La ressource de base du jeu utilisée pour les améliorations, attaques, déplacements et réparations |
| **DFT** | Dark Forest Token, un token ERC-20 utilisé pour les sauts, améliorations, amendes de sortie d'alliance, etc. |
| **Coordonnées** | Position d'une civilisation dans l'espace 3D `(x, y, z)`, attribuée via un algorithme déterministe |
| **Niveau total** | La somme de tous les niveaux de systèmes ; affecte les coûts d'entretien quotidiens |
| **Portée de scan** | Votre rayon de détection : `1000 + 150×nv + 5×nv²` ; détermine qui vous pouvez attaquer ou poursuivre |
| **Puissance d'attaque** | `900 + 10 × nv²` ; l'attribut principal du système d'arme |
| **Défense** | `540 + 6 × nv²` ; l'attribut principal du système de bouclier |
| **PV du bouclier** | `3600 + 15 × nv²` ; absorbe les dégâts entrants |
| **Vitesse moteur** | `10 + 5 × nv` ; vitesse de croisière |
| **Taux de collecte** | `3 + 10 × √(nv - 1)` énergie/seconde |
| **Durabilité** | État de fonctionnement de chaque système, consommé avec le temps ou par l'utilisation |
| **Entretien quotidien** | `2 000 × nv total / jour` ; déduit proportionnellement au fil du temps |
| **Frais d'entrée** | BNB payé lors de la création d'une civilisation, croissance linéaire de 0,01 à 0,05 BNB |
| **Formule d'ancrage** | Formule de calcul des coûts d'amélioration basée sur la production quotidienne et les jours d'ancrage |
| **Jours d'ancrage** | Niveau 1–100 : 1–30 jours ; 100–1000 : 30–60 jours ; 1000+ : 60 jours (plafonné) |
| **Croisière** | Méthode de déplacement standard — se déplace vers des coordonnées cibles à vitesse constante |
| **Saut spatial** | Téléportation instantanée vers de nouvelles coordonnées, coûte énergie + DFT, bouclier réduit à 10 % |
| **Saut de poursuite** | Saut instantané vers l'emplacement d'une cible ; nécessite Radar Nv20+ et cible à portée |
| **Token d'attaque** | Algorithme de seau de tokens limitant la fréquence d'attaque ; max 3–10 tokens |
| **Ratio de pillage** | `5 % + nv d'arme × 0,5 %` ; portion de l'énergie cible pillée en cas de victoire |
| **Protection des nouveaux joueurs** | Impossible d'être attaqué pendant les premières 24 heures après la création |
| **Ruines** | Une civilisation qui ne peut plus être rétrogradée après une défaite ; peut être ravivée via Reconstruction |
| **Reconstruction / Renaissance** | Ravive une civilisation en ruines, coûte énergie + DFT, accorde un bonus de combat |
| **Bonus de combat** | +12 % par reconstruction, max 300 % (30000 bps) |
| **Alliance** | Organisation de joueurs pour la collaboration, max 100 membres |
| **Délai de refroidissement de sortie** | Période de non-agression mutuelle de 24 heures après avoir quitté une alliance |
| **Totem** | Système de buff à l'échelle de l'alliance ; donner de l'énergie pour l'améliorer pour des bonus de défense |
| **Époque** | Unité de temps du DailyMinter, de minuit UTC au prochain minuit UTC |
| **DELEGATECALL** | Architecture de contrat où le contrat principal délègue la logique aux contrats satellites |
| **Porte douce** | Quand les conditions d'attaque ne sont pas remplies, le jeu montre la raison sans consommer de ressources |
