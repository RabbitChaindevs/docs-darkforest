# Marché de l'énergie

Le marché de l'énergie est un **carnet d'ordres décentralisé** où les joueurs peuvent échanger de l'énergie contre du DFT. Il est alimenté par le contrat `EnergyMarket.sol`.

---

## Vendre de l'énergie (Créer un ordre)

### Opération

```
createOrder(montantÉnergie, prixDFT)
```

1. Entrez la **quantité d'énergie** que vous voulez vendre
2. Entrez le **prix total en DFT** que vous demandez
3. Confirmez la transaction

### Verrouillage de l'énergie

Lorsque vous créez un ordre, le contrat **verrouille immédiatement** votre énergie (marquée comme `energyReserved`), vous empêchant de la dépenser tant que l'ordre est actif.

### Annuler un ordre

```
cancelOrder(idOrdre)
```

- L'ordre doit avoir au moins **3 secondes** d'ancienneté avant l'annulation (empêche le frontrunning dans le même bloc)
- Après l'annulation, l'énergie est déverrouillée

---

## Acheter de l'énergie (Exécuter un ordre)

### Opération

```
fillOrder(idOrdre, prixMax)
```

1. Parcourez la liste des ordres de vente actifs
2. Sélectionnez un ordre
3. Entrez le prix maximum que vous êtes prêt à accepter
4. Confirmez la transaction

### Frais

Des **frais de protocole DFT de 1 %** sont déduits du produit du vendeur :

```
L'acheteur paie = prix de l'ordre (prix de liste exact)
Le vendeur reçoit = prix de l'ordre × 99 %
Frais de protocole = prix de l'ordre × 1 % (accumulés dans le contrat)
```

Les frais de protocole de 1 % s'accumulent dans le contrat et peuvent être retirés par l'administrateur.

---

## Paramètres du marché

| Paramètre | Valeur |
|-----------|--------|
| Frais de protocole | 1 % (déduit du vendeur) |
| Âge minimum de l'ordre | 3 secondes (avant annulation/exécution) |
| Statut de l'ordre | actif / inactif (devient inactif après annulation ou exécution) |

---

## Exemple de transaction

### Alice vend de l'énergie

1. Alice crée un ordre de vente : `createOrder(10000, 100 DFT)` — vend 10 000 énergie pour un total de 100 DFT
2. Le contrat verrouille les 10 000 énergie d'Alice

### Bob achète de l'énergie

1. Bob voit l'ordre d'Alice
2. Appelle `fillOrder(orderId, 100)` — paie exactement 100 DFT
3. Alice reçoit 99 DFT (100 - 1 % de frais)
4. 1 DFT est accumulé comme frais de protocole

---

## Conseils stratégiques

| Scénario | Suggestion |
|----------|------------|
| Besoin urgent d'énergie pour le combat | Acceptez les ordres de vente les moins chers |
| Pas pressé | Placez un ordre limité et attendez le bon prix |
| Le prix du DFT augmente | Vendez de l'énergie pour du DFT pour profiter |
| Excédent d'énergie | Listez des ordres de vente pour gagner du DFT |
