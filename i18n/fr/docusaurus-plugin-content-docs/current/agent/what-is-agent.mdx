# Qu'est-ce qu'un agent IA

Un agent IA est un **programme automatisé hors chaîne** qui effectue des opérations de jeu au nom de votre civilisation. Il est autorisé via un **masque de permission** et opère dans le cadre de **contraintes de politique**.

---

## Ce qu'un agent peut faire

Les capacités de l'agent sont contrôlées par des **permissions autorisées** :

| Permission | Valeur | Action |
|-----------|--------|--------|
| COLLECT | 1 | Collecter l'énergie |
| REPAIR | 2 | Réparer les systèmes |
| CLAIM_COMBAT | 8 | Réclamer l'énergie de combat |
| ATTACK | 32 | Attaquer |
| UPGRADE | 64 | Améliorer les systèmes |
| JUMP | 128 | Saut spatial |
| REBUILD | 256 | Reconstruire la civilisation |
| DONATE | 512 | Donner au totem |

Vous pouvez accorder n'importe quelle combinaison de permissions à un agent (masque binaire).

---

## Contraintes de politique

En plus des permissions, vous pouvez définir des **contraintes de politique** pour l'agent :

| Contrainte | Description |
|-----------|-------------|
| `minEnergyReserve` | Réserve d'énergie minimale (l'agent ne dépensera pas en dessous) |
| `minDftReserve` | Réserve minimale de DFT |
| `maxAttackPerHour` | Attaques maximales par heure |
| `maxUpgradePerDay` | Améliorations maximales par jour |
| `maxSingleUpgradeCost` | Coût DFT maximal par amélioration |
| `autoCollect` | Collecte automatique d'énergie |
| `autoRepair` | Réparation automatique des systèmes |
| `autoShieldRepair` | Réparation automatique du bouclier |
| `autoUpgrade` | Amélioration automatique |
| `autoAttack` | Attaque automatique |

## Mécanismes de sécurité

### 1. Enregistrement on-chain

La politique de l'agent est stockée sur le contrat `DarkForestAgentRegistry.sol` et ne peut pas être falsifiée une fois définie.

### 2. Séparation des permissions

- **Autoriser** : Vous autorisez l'agent à agir en votre nom
- **Masque de permission** : Contrôle fin sur chaque opération
- **Révocable** : `revokeAgent()` prend effet immédiatement

### 3. Disjoncteurs

```
Limite de fréquence d'attaque : max N par heure
Limite de fréquence d'amélioration : max N par jour
Plancher d'énergie : jamais en dessous de X
Plancher de DFT : jamais en dessous de Y
```

### 4. Liste noire / Liste blanche

- **Liste noire** : Interdire l'attaque de joueurs spécifiques
- **Liste blanche** : Autoriser uniquement l'attaque de joueurs spécifiques (la liste noire est ignorée quand la liste blanche est active)

## Agent vs Jeu manuel

| Comparaison | Manuel | Agent IA |
|------------|--------|----------|
| Présence en ligne | Doit être souvent en ligne | Fonctionnement 24/7 |
| Temps de réponse | Latence | Instantané |
| Précision | Sujet aux erreurs | Suit strictement la politique |
| Sécurité | Clé privée dans le portefeuille | Stockage chiffré + contraintes on-chain |
