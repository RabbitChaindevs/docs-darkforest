# Modèles de stratégie

Les stratégies d'agent sont définies à l'aide de fichiers de configuration YAML.

---

## Structure du fichier de stratégie

```yaml
agent:
  name: "mon-agent"
  mode: "auto"          # auto | manual | dry-run

limits:                 # Disjoncteur (politique on-chain)
  min_energy_reserve: 100000
  min_dft_reserve: 10
  max_attacks_per_hour: 5
  max_upgrades_per_day: 3
  max_single_upgrade_cost: 100

combat:                 # Stratégie de combat
  min_target_level: 1
  max_target_level: 999
  weapon_level: 1
  only_attack_when_energy_above: 100000
  blacklist: []         # Adresses en liste noire
  whitelist: []         # Adresses en liste blanche (attaquer seulement celles-ci)

defense:                # Stratégie de défense
  auto_repair: true
  min_durability_before_repair: 20
  shield_regen_threshold: 50   # Démarrer régén. bouclier en dessous de ce %

collection:             # Stratégie de collecte
  enabled: true
  interval_minutes: 60
```

---

## Quatre stratégies prédéfinies

### 1. Agressive

```yaml
combat:
  max_attacks_per_hour: 10
  min_target_level: 0.5
  weapon_level: max

collection:
  enabled: false  # Pas de collecte, combat pur
```

- ✅ Bon pour : joueurs très actifs qui veulent des récompenses de combat
- ❌ Risque : consommation d'énergie élevée, risque de contre-attaque

### 2. Équilibrée

```yaml
combat:
  max_attacks_per_hour: 5
  min_target_level: 0.8
  weapon_level: balanced

collection:
  enabled: true
  interval_minutes: 30

defense:
  auto_repair: true
  shield_regen_threshold: 60
```

- ✅ **Recommandée** : attaque et défense équilibrées, convient à la plupart des joueurs

### 3. Défensive

```yaml
combat:
  max_attacks_per_hour: 1
  only_counterattack: true  # Contre-attaquer uniquement

defense:
  auto_repair: true
  min_durability_before_repair: 50
  shield_regen_threshold: 80

collection:
  enabled: true
  interval_minutes: 15
```

- ✅ Bon pour : joueurs hors ligne pendant de longues périodes qui veulent éviter les risques

### 4. Fermier

```yaml
combat:
  enabled: false  # Pas de combat

defense:
  auto_repair: true
  shield_regen_threshold: 90

collection:
  enabled: true
  interval_minutes: 5
```

- ✅ Bon pour : développement en début de jeu et accumulation de ressources

## Stratégie personnalisée

```bash
# Copier le modèle
cp templates/balanced.yaml ma-strategie.yaml

# Modifier les paramètres
vim ma-strategie.yaml

# Utiliser la stratégie personnalisée
python -m dark_forest.cli run --strategy ./ma-strategie.yaml
```

---

## Paramètres du disjoncteur

| Paramètre | Description | Défaut |
|-----------|-------------|--------|
| `max_attacks_per_hour` | Attaques max par heure | 10 |
| `min_energy_reserve` | Réserve d'énergie minimale | 100 000 |
| `min_dft_reserve` | Réserve DFT minimale | 10 |
| `max_upgrades_per_day` | Améliorations max par jour | 5 |

> ⚠️ Les paramètres du disjoncteur sont appliqués on-chain. L'agent ne peut pas les contourner.
