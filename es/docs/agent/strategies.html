Plantillas de Estrategia

Las estrategias del agente se definen usando archivos de configuración YAML.

Estructura del Archivo de Estrategia​

agent:
  name: "mi-agente"
  mode: "auto"          # auto | manual | dry-run

limits:                 # Interruptor de seguridad (política en cadena)
  min_energy_reserve: 100000
  min_dft_reserve: 10
  max_attacks_per_hour: 5
  max_upgrades_per_day: 3
  max_single_upgrade_cost: 100

combat:                 # Estrategia de combate
  min_target_level: 1
  max_target_level: 999
  weapon_level: 1
  only_attack_when_energy_above: 100000
  blacklist: []         # Direcciones en lista negra
  whitelist: []         # Direcciones en lista blanca (solo atacar estas)

defense:                # Estrategia de defensa
  auto_repair: true
  min_durability_before_repair: 20
  shield_regen_threshold: 50   # Iniciar regeneración del escudo por debajo de este %

collection:             # Estrategia de recolección
  enabled: true
  interval_minutes: 60

Cuatro Estrategias Predefinidas​

1. Agresiva​

combat:
  max_attacks_per_hour: 10
  min_target_level: 0.5
  weapon_level: max

collection:
  enabled: false  # Sin recolección, combate puro
  • ✅ Bueno para: jugadores muy activos que quieren recompensas de combate
  • ❌ Riesgo: alto consumo de energía, riesgo de ser contraatacado

2. Equilibrada​

combat:
  max_attacks_per_hour: 5
  min_target_level: 0.8
  weapon_level: balanced

collection:
  enabled: true
  interval_minutes: 30

defense:
  auto_repair: true
  shield_regen_threshold: 60
  • ✅ Recomendada: ataque y defensa equilibrados, adecuada para la mayoría de jugadores

3. Defensiva​

combat:
  max_attacks_per_hour: 1
  only_counterattack: true  # Solo contraatacar

defense:
  auto_repair: true
  min_durability_before_repair: 50
  shield_regen_threshold: 80

collection:
  enabled: true
  interval_minutes: 15
  • ✅ Bueno para: jugadores que están desconectados por largos períodos y quieren evitar riesgos

4. Recolectora​

combat:
  enabled: false  # Sin combate

defense:
  auto_repair: true
  shield_regen_threshold: 90

collection:
  enabled: true
  interval_minutes: 5
  • ✅ Bueno para: desarrollo en etapas tempranas y acumulación de recursos

Estrategia Personalizada​

# Copiar la plantilla
cp templates/balanced.yaml mi-estrategia.yaml

# Editar parámetros
vim mi-estrategia.yaml

# Usar estrategia personalizada
python -m dark_forest.cli run --strategy ./mi-estrategia.yaml

Parámetros del Interruptor de Seguridad​

ParámetroDescripciónValor por Defecto
max_attacks_per_hourMáx. ataques por hora10
min_energy_reserveReserva mínima de energía100.000
min_dft_reserveReserva mínima de DFT10
max_upgrades_per_dayMáx. mejoras por día5

⚠️ Los parámetros del interruptor de seguridad se aplican en cadena. El agente no puede omitirlos.
