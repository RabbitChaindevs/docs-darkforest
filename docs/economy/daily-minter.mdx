# 每日铸造

**DailyMinter** 合约每天铸造一定数量的 DFT，按活跃玩家数量平均分配。**当天不领就没了**，不累积。

---

## 如何领取

### 条件

- 你必须在领取当天的 UTC **午夜之前**已经创建了文明
- 当天必须有 `distribute()` 被调用过（任何人都可以触发）
- 每个地址每天只能领取一次

### 操作

```
claim()
```

确认交易（只需 Gas 费）。

---

## 铸造流程

### 第 1 步：分发（Distribute）

任何人都可以调用 `distribute()`：

1. 合约检查上一个未分发的 epoch
2. 快照当前活跃文明数量
3. 计算 `每人份额 = 每日排放量 / 活跃文明数`
4. 铸造总 DFT 到合约
5. `每日排放量 = 1,152,575,342 DFT`（常量）

### 第 2 步：领取（Claim）

玩家调用 `claim()`：

1. 验证你在此 epoch 前已创建文明
2. 验证你尚未领取过
3. 将你的份额转入你的钱包

### 第 3 步：销毁剩余

下一次 `distribute()` 时，上一个 epoch 未被领取的 DFT 会被**销毁**（减少流通量）。

---

## 时间线

```
Epoch 0      Epoch 1      Epoch 2
   |            |            |
UTC 00:00    UTC 00:00    UTC 00:00
   ↑创建文明     ↑distribute   ↑distribute
                ↑claim        ↑claim
                              ↑未领的销毁
```

- Epoch 按 UTC 午夜划分
- 你只能领取**最新已分发**的 epoch
- 错过 epoch 就无法补领

---

## 关键参数

| 参数 | 值 |
|------|-----|
| 每日排放 | **1,152,575,342 DFT**（常量） |
| 总分配期 | **3,650 天**（约 10 年） |
| 总供应上限 | **420,690,000,000,000 DFT** |
| 领取窗口 | 每个 epoch 内（通常是 1 天） |

---

## 为什么每天要领取

1. 🆓 **免费 DFT** — 只需 Gas 费，每天都能领
2. 📉 **不领就销毁** — 错过了就没了
3. 📈 **玩家越少，每人分得越多** — 如果活跃玩家减少，每人份额会上升

> 💡 建议设置每日提醒 UTC 时间领取。如果当天 `distribute()` 还没被调用，你也可以自己调用它——只需要 Gas 费。
