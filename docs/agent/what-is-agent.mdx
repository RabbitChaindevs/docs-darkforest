# 什么是 AI Agent

AI Agent 是一个**链下自动化程序**，代表你的文明自动执行游戏操作。Agent 通过**权限位掩码**获得你的授权，在**策略约束**范围内操作。

---

## Agent 能做什么

Agent 通过**授权权限**控制可执行的操作：

| 权限位 | 值 | 操作 |
|--------|-----|------|
| COLLECT | 1 | 采集能源 |
| REPAIR | 2 | 修复系统 |
| CLAIM_COMBAT | 8 | 领取战斗能源 |
| ATTACK | 32 | 攻击 |
| UPGRADE | 64 | 升级系统 |
| JUMP | 128 | 空间跳跃 |
| REBUILD | 256 | 重建文明 |
| DONATE | 512 | 捐赠图腾 |

你可以授予 Agent 任意权限组合（位掩码）。

---

## 策略约束（Policy）

除了权限控制，你还可以为 Agent 设置**策略约束**：

| 约束 | 说明 |
|------|------|
| `minEnergyReserve` | 最低能源保留量（Agent 不会消耗到低于此值） |
| `minDftReserve` | 最低 DFT 保留量 |
| `maxAttackPerHour` | 每小时最大攻击次数 |
| `maxUpgradePerDay` | 每天最大升级次数 |
| `maxSingleUpgradeCost` | 单次升级最大 DFT 消耗 |
| `autoCollect` | 是否自动采集 |
| `autoRepair` | 是否自动修复 |
| `autoShieldRepair` | 是否自动修复护盾 |
| `autoUpgrade` | 是否自动升级 |
| `autoAttack` | 是否自动攻击 |

## 安全机制

### 1. 链上注册

Agent 的策略存储在 `DarkForestAgentRegistry.sol` 合约中，一旦设置不可篡改。

### 2. 权限分离

- **授权（Authorize）**：你授权 Agent 代你执行操作
- **权限位掩码**：精确控制每个操作
- **可随时撤销**：`revokeAgent()` 立即失效

### 3. 熔断限制

```
攻击频率限制：每小时最多 N 次
升级频率限制：每天最多 N 次
能源保留底线：不低于 X
DFT 保留底线：不低于 Y
```

### 4. 黑名单/白名单

- **黑名单**：禁止攻击指定玩家
- **白名单**：只允许攻击指定玩家（开启白名单后黑名单失效）

## 两阶段架构

```
┌─────────────────────────────────────┐
│          链下 Agent                 │
│  ┌──────────────────────────────┐   │
│  │ 策略引擎（YAML 配置文件）      │   │
│  │ 四种预设策略                  │   │
│  │ 自定义策略模板                │   │
│  └──────────┬───────────────────┘   │
│             │ 通过合约调用          │
│             ▼                      │
│  ┌──────────────────────────────┐   │
│  │ Agent SDK                    │   │
│  │ 调用合约方法（只读/写入）      │   │
│  └──────────────────────────────┘   │
└──────────────┬──────────────────────┘
               │
               ▼
┌─────────────────────────────────────┐
│    链上 DarkForestAgentRegistry     │
│  ┌──────────────────────────────┐   │
│  │ 权限位掩码检查               │   │
│  │ 策略约束检查                 │   │
│  │ 频率限制检查                 │   │
│  └──────────────────────────────┘   │
└─────────────────────────────────────┘
```

## Agent vs 手动操作

| 对比 | 手动操作 | AI Agent |
|------|---------|----------|
| 在线要求 | 需要经常在线 | 24/7 运行 |
| 响应速度 | 有延迟 | 即时 |
| 操作精度 | 可能出错 | 严格按策略 |
| 安全 | 私钥在钱包 | 加密存储 + 链上约束 |
